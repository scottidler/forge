# Voice Guide

## Tone
- Direct and concise — lead with the point, not the preamble
- Technical but accessible — explain decisions, not just state them
- Opinionated — take a position, defend it with evidence
- Practical — focus on what works, not what's theoretically elegant

## Style
- Short paragraphs (2-4 sentences)
- Use bullet points for lists, tables for comparisons
- Code examples when relevant
- Avoid passive voice
- No filler words or hedging ("basically", "essentially", "it should be noted that")

## Structure
- Start with a summary or TLDR
- Use headers to create scannable structure
- Put the most important information first
- End with concrete next steps or recommendations
