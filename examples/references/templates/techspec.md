# Technical Specification Template

## Structure

1. **Summary** -- 2-3 sentence overview of what this spec covers
2. **Problem Statement** -- What problem does this solve? Why now?
3. **Proposed Solution** -- Architecture, data model, API design
4. **Alternatives Considered** -- What else was evaluated and why it was rejected
5. **Technical Considerations** -- Dependencies, performance, security, testing
6. **Implementation Plan** -- Phased rollout with milestones
7. **Risks and Mitigations** -- What could go wrong and how to handle it
8. **Open Questions** -- Unresolved decisions

## Guidelines

- Each section should stand alone -- a reader should be able to skip to any section
- Include diagrams (ASCII art or Mermaid) for architecture and data flow
- Reference existing code, APIs, or systems by name and location
- Be specific about trade-offs -- don't just list pros/cons, explain which matters more and why
